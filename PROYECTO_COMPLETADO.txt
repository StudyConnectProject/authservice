🔌 API ENDPOINTS
══════════════════════════════════════════════════════════════════════════════════

Health Check:
  GET /api/v1/auth/health
  → Retorna: "Auth Service is running"

Registro:
  POST /api/v1/auth/register
  Body: { email, password, role }
  Response: { access_token, refresh_token, user }

Login:
  POST /api/v1/auth/login
  Body: { email, password, ip_address? }
  Response: { access_token, refresh_token, user }

Refresh:
  POST /api/v1/auth/refresh
  Body: { refresh_token, ip_address? }
  Response: { access_token, refresh_token, user }

Logout:
  POST /api/v1/auth/logout/{userId}
  Response: 204 No Content


🗄️  BASE DE DATOS SCHEMA
══════════════════════════════════════════════════════════════════════════════════

USERS
├── id (UUID PRIMARY KEY)
├── email (VARCHAR 255 UNIQUE NOT NULL)
├── password_hash (VARCHAR 255)
├── is_active (BOOLEAN DEFAULT true)
└── created_at (TIMESTAMP DEFAULT NOW)

ROLES
├── id (UUID PRIMARY KEY)
├── name (VARCHAR 50 UNIQUE) - STUDENT, TUTOR
├── is_active (BOOLEAN DEFAULT true)
└── created_at (TIMESTAMP)

USER_ROLES
├── id (UUID PRIMARY KEY)
├── user_id (UUID FK)
├── role_id (UUID FK)
├── assigned_at (TIMESTAMP)
└── UNIQUE(user_id, role_id)

REFRESH_TOKENS
├── id (UUID PRIMARY KEY)
├── user_id (UUID FK)
├── token_hash (VARCHAR 255)
├── ip_address (VARCHAR 45)
├── is_revoked (BOOLEAN)
├── expires_at (TIMESTAMP)
└── created_at (TIMESTAMP)

🚦 INICIO RÁPIDO
══════════════════════════════════════════════════════════════════════════════════

1. INICIAR SERVICIOS:

   Linux/Mac:
   $ chmod +x start.sh
   $ ./start.sh

   Windows:
   > start.bat

   O manualmente:
   $ docker-compose up -d

2. ESPERAR INICIALIZACIÓN:
   
   ⏳ 30-40 segundos para que PostgreSQL esté listo
   
   Verificar estado:
   $ docker-compose ps
   
   Ver logs:
   $ docker-compose logs -f authservice

3. TESTING:

   Health check:
   $ curl http://localhost:8080/api/v1/auth/health
   
   O importar Postman_Collection.json en Postman

4. DETENER:

   Linux/Mac:
   $ ./stop.sh

   Windows:
   > stop.bat

   O:
   $ docker-compose down -v

📞 SOPORTE
══════════════════════════════════════════════════════════════════════════════════

Problemas comunes:

❌ "Connection refused"
   → Esperar a que PostgreSQL se inicie (30-40 seg)
   → Ver: docker-compose logs postgres

❌ "Port already in use"
   → Cambiar puerto en docker-compose.yml
   → Ejemplo: 5433:5432 en lugar de 5432:5432

❌ "Email already exists"
   → Usar otro email o limpiar BD
   → Ver TESTING_GUIDE.md

❌ "Invalid token"
   → Generar nuevo token con login
   → Verificar JWT_SECRET
